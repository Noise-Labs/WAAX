Feature Highlights

  • Modular and extensible architecture
  • Web Audio Plug-in (WAPL) Interface
  • Modular GUI elements powered by Polymer
  • Sequencing: Timebase, Transport and NoteList
  • Web MIDI API Integration (experimental)
  • Create, build and deploy: Bower and Gulp

What is WAAX?

WAAX (Web Audio API eXtension) is a comprehensive JavaScript framework that facilitates and supports the development of web music applications as well as various collaborative music creation scenarios.

At the moment, WAAX and MUI require Blink-based web browsers - Chrome, Chrome Canary or Chromium - because they are built on top of Web Audio API, Web Component and Web MIDI API.

Get Started

Install.

The complete WAAX development setup requires NPM, Bower and Gulp. If you have them installed, the installation is simple.

git clone https://github.com/hoch/WAAX $YOUR_DIRECTORY
cd $YOUR_DIRECTORY
make

Load.

To use WAAX in a web page, simply load the waax.js in an HTML document. You can also add WAAX plug-ins that you want to play with.

<!-- load WAAX, MUI, Timebase and plug-ins -->
<script src="build/waax.js"></script>
<script src="build/plug_ins/wxs1.js"></script>
<script src="build/plug_ins/stereodelay.js"></script>

Play.

Once you load the core library and plug-ins. Using them is straight-forward and does not ask you to have audio hacking experience. Just create instances and path them to make sounds. NOTE: Adjust your audio volume before you do this step!

// Creating a WXS-1 synth and a stereo delay instance.
var synth = WX.WXS1(),
    delay = WX.StereoDelay();
// Plug-in patching: Synth => Delay => Master
synth.to(delay).to(WX.Master);
// Plays C4 with 100 velocity for 1 second.
synth.noteOn(60, 100);
synth.noteOff(WX.now + 1.0);

Iterate. Fast.

Then start rolling wit Gulp! WAAX offers basic Gulp commands so you can iterate and deploy rapidly.

gulp build           # generates distribution package
gulp deploy          # builds and deploys to gh-pages
gulp                 # cleans, builds and starts dev server
